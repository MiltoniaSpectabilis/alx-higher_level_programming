TDD is an approach that consists of creating tests before building code that conforms to them. This project is a basic application of TDD.

All the tests in the "tests" directory were drafted prior to the code. Testing is done using the doctest module.

The goal of the codes should be obvious, but for the sake of explaining:
0-add_integer: addition function
2-matrix_divided: divides each element in a matrix by the divisor
3-say_my_name: Heisenberg? you're goddamn right
4-print_square: pritns a square of size (size - arg) using #

Columns (:) are used to separate the tests for better readability
